# E2E 测试指南

本项目使用 Playwright 进行端到端测试，覆盖核心用户场景和功能。

## 测试环境设置

### 安装依赖

```bash
npm install
```

### 安装 Playwright 浏览器

```bash
npx playwright install
```

## 运行测试

### 运行所有测试

```bash
npm run test:e2e
```

### 运行特定测试文件

```bash
npx playwright test auth.spec.ts
```

### 运行特定浏览器测试

```bash
# Chromium
npx playwright test --project=chromium

# Firefox
npx playwright test --project=firefox

# WebKit (Safari)
npx playwright test --project=webkit

# 移动端
npx playwright test --project=mobile

# 平板
npx playwright test --project=tablet
```

### 交互式UI模式

```bash
npm run test:e2e:ui
```

### 调试模式

```bash
npm run test:e2e:debug
```

### 有头模式（查看浏览器）

```bash
npm run test:e2e:headed
```

## 查看测试报告

```bash
npm run test:e2e:report
```

测试报告会自动生成在 `playwright-report` 目录中。

## 测试覆盖范围

- **桌面浏览器：**

  - WebKit (Safari)

- **移动设备：**
  - iPhone 13
  - iPad Pro

测试覆盖三种设备类型：
- **桌面端：** 1280x720
- **平板端：** 768x1024
- **移动端：** 375x812

## 测试数据

测试使用动态生成的测试数据，每次运行都会创建新的用户、项目、任务和风险，避免数据冲突。

## 测试报告

测试完成后会生成以下报告：

1. **HTML报告：** `playwright-report/index.html`
2. **JSON报告：** `test-results/results.json`
3. **截图：** 失败测试的截图保存在 `test-results/` 目录
4. **视频：** 失败测试的视频录制保存在 `test-results/` 目录
5. **Trace：** 失败测试的trace文件保存在 `test-results/` 目录

## 最佳实践

### 1. 测试隔离

每个测试都独立运行，使用 `beforeEach` 钩子确保测试环境干净。

### 2. 等待策略

使用 Playwright 的自动等待机制，避免硬编码的等待时间。

### 3. 选择器策略

使用多种选择器策略，提高测试的健壮性：
- 文本选择器：`text=登录`
- CSS选择器：`.ant-button`
- 属性选择器：`[data-testid="login-button"]`
- 组合选择器：`button:has-text("登录")`

### 4. 辅助函数

使用辅助函数封装常用操作，减少代码重复：
- `registerAndLogin()`: 注册并登录用户
- `createProject()`: 创建项目
- `createTask()`: 创建任务
- `createRisk()`: 创建风险

## 故障排查

### 测试失败

1. 查看测试报告：`npm run test:e2e:report`
2. 查看截图和视频：`test-results/` 目录
3. 使用调试模式：`npm run test:e2e:debug`

### 浏览器问题

```bash
# 重新安装浏览器
npx playwright install --force
```

### 网络问题

测试配置了 `ignoreHTTPSErrors: true`，可以忽略HTTPS证书错误。

## CI/CD 集成

在 CI/CD 环境中，测试会：
- 自动重试失败的测试（2次）
- 限制并行workers数量
- 生成详细的测试报告
- 上传测试产物

## 维护建议

1. **定期更新依赖：** 保持 Playwright 版本最新
2. **维护测试数据：** 定期检查测试数据生成逻辑
3. **更新选择器：** 当UI变化时，及时更新测试选择器
4. **添加新测试：** 新功能开发时，同步添加E2E测试
5. **优化性能：** 定期检查测试执行时间，优化慢速测试
